# Верифікація каси (Vite + React PWA, IndexedDB)

## Запуск

1. `npm run dev` у фоні → http://localhost:5173 (готовий за ~1 с).
2. Пароль-гейт активний і в dev (Vite читає `.env` → `VITE_APP_PASSWORD`) — пароль дивись у `.env`.

## Драйвер браузера

Playwright/puppeteer у проєкті немає. Робочий рецепт: `puppeteer-core` (ставиться за ~6 с у scratchpad, без завантаження браузера) + системний Edge:

```js
puppeteer.launch({ executablePath: 'C:/Program Files (x86)/Microsoft/Edge/Application/msedge.exe', headless: 'new' })
```

Кожен запуск — свіжий тимчасовий профіль → чиста IndexedDB → прогін завжди починається з екрана першого запуску (точка + касири).

## Підводні камені

- **`innerText` віддає текст після CSS `text-transform: uppercase`** (заголовки типу «ХТО ВІДКРИВАЄ ЗМІНУ?») — перевірки тексту роби регістронезалежними.
- Клік по тексту кнопки: `[...document.querySelectorAll(sel)].find(e => e.textContent.includes(txt)).click()` через `page.evaluate` — селекторів з data-testid у проєкті немає.
- `favicon.ico` дає 404 у dev — відомий шум, не пов'язаний зі змінами.
- Час подій (чеки, зміни) — реальний `Date.now()`; для сценаріїв «вчорашня зміна» редагуй IndexedDB напряму через `page.evaluate` (store `shifts`, БД `kasa-db` версія 2) і перезавантажуй сторінку.
- Щоб підготувати IndexedDB **до** запуску коду застосунку (сценарії міграції БД): поклади тимчасовий `public/_blank.html`, зайди на нього (той самий origin), створи базу через `page.evaluate`, потім переходь на `/`. Навігація на `/catalog.csv` не працює (браузер скачує файл), `setJavaScriptEnabled(false)` ламає async `evaluate`.

## Що ганяти (основний цикл)

Перший запуск (точка+касири) → відкриття зміни → додати товар у кошик → «До оплати» → «Прийняти оплату» → Журнал (чек «М-N», сторно з причиною) → «Закрити зміну» (модалка з підсумком) → повернення на екран відкриття зміни.
